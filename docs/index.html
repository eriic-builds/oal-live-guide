Set up an Organization Asset Library for PowerPoint templates

00 Overview

An organization assets library is a SharePoint document library you promote so its files surface across Microsoft 365. It can hold different asset types — for branded decks you want the Office templates type, which is all this guide covers.

Office templates this guide

When a user creates a new Office document, they see a tab for your organization with the available templates. Supported in Word, Excel, and PowerPoint.

Images & logos

Photos and logos that appear under "Your organization" in the SharePoint file picker (and "Brand Images" in PowerPoint). A separate setup — not covered here.

Key parameter For templates, the library must be registered with -OrgAssetType OfficeTemplateLibrary. If you omit the type it defaults to an image library — the wrong one for templates.

02 Build the SharePoint site from scratch

This is the foundation. An organization assets library is just an ordinary SharePoint document library that you later promote with PowerShell — so first you stand up a normal site and library, then bridge it into an OAL. You do not need Brand Central. Source: Create an organization assets library.

A. Create a new site

  1. Open the create flow. Go to https://contoso.sharepoint.com and click + Create site on the SharePoint start page. (Admins can also use SharePoint admin center → Active sites → + Create.)
  2. Choose a site type. Pick Communication site. Any modern site works — communication, a Microsoft 365 Group team site, or a non-group team site — but a communication site fits the "few publishers, everyone reads" brand-asset pattern and doesn't spin up a Group you don't need.
  3. Pick a design (optional). The Topic or Blank template is fine — the site design has no effect on OAL behavior; it only styles the landing page.
  4. Name & address it. Site name e.g. Brand Templates. The site address becomes the URL path — e.g. /sites/branding → https://contoso.sharepoint.com/sites/branding. Note this URL; you'll build on it.
  5. Set language & owner. Choose the default language (can't be changed later), set yourself as primary administrator, then Finish. The site provisions in a few seconds.

B. Create the document library

  1. Add a library. On the new site, click + New → Document library → Blank library.
  2. Name it. e.g. Templates (and an optional description). Click Create. Its URL becomes https://contoso.sharepoint.com/sites/branding/Templates — this exact library URL is what you register in §06.
  3. It must be a library, not a folder. The OAL designation only accepts document libraries. If you want sub-groupings, use folders inside this one library — don't make separate libraries expecting each to be its own OAL unless you intend to register each.

C. Set permissions correctly

  1. Publishers = Members/Owners. ⚙ Settings → Site permissions. Add the people who will upload and maintain templates as Members (edit) or Owners (full control).
  2. Readers = "Everyone except external users" (REQUIRED). This group must have Read on the library before you register — missing it is the #1 reason Add-SPOOrgAssetsLibrary fails (see the error callout below). Exact path that works: on the Templates site, click ⚙ Settings → Site permissions → Share site, add "Everyone except external users", and set the permission level to Read (not Edit). Then save.
  3. Fine-tune if needed. You can customize permissions on up to 100 files/folders in the library (e.g. to hold back a draft template) without breaking the OAL.
If you hit this error Add-SPOOrgAssetsLibrary : "Everyone except external users" needs to be given read permission to the document library. It means the read grant wasn't applied. Fix: on the Templates site go to ⚙ Settings → Site permissions → Share site, add "Everyone except external users" set to Read, then re-run the §06 command. aka.ms/orgassets.
Bridge — site vs. OAL Creating this site and library does not make it an organization assets library yet. Right now it's a normal SharePoint library. The "OAL" is a designation you apply later with Add-SPOOrgAssetsLibrary (§06), which is what surfaces the templates under your org's tab in PowerPoint and enables a delivery CDN. §03 fills the library; §04–06 perform the promotion.
Up to 30 OAL libraries per org All OAL libraries on the same site Designate libraries, not folders

03 Build, save & upload your .potx templates

⚠ The file MUST be a .potx The Organization Asset Library supports PowerPoint templates in .potx format. A standard .pptx presentation is not a supported organizational template format and won't appear in the PowerPoint organizational templates experience. Always save and upload PowerPoint templates as .potx. Source: supported template formats.

A template is a deck whose slide master and layouts carry your branding, saved in the .potx format — the only PowerPoint format the OAL accepts — then uploaded to your library. Source: Create and save a PowerPoint template.

Already have a template? Skip to C If you already have a finished .potx, just upload it (card C) — no need to rebuild. Have a polished .pptx instead? Open it and do a one-time File → Save As → PowerPoint Template (*.potx) (card B), then upload. Card A is only for authoring a brand-new template from scratch.

A. Build the template (new templates only)

  1. Start a blank deck. File → New → Blank Presentation.
  2. Edit the slide master. View → Slide Master, then apply your brand via Themes, Background Styles, and Insert Placeholder on the master and its layouts. (Modifying the master/layouts is what makes it a reusable template.)

B. Save as .potx

  1. Save As. File → Save As (or Save a Copy in Microsoft 365).
  2. Choose the template type. In the Save as type list, select PowerPoint Template — PowerPoint switches the location to your Custom Office Templates folder. Name it and Save.

C. Upload to the library

  1. Upload the .potx. Put the .potx file(s) into the Templates library from §02. Double-check the extension reads .potx, not .pptx before you register. This library is what you'll register next.
  2. (Optional) Add a thumbnail image. For a custom card background in the file picker, place an image (e.g. your logo) on the same site and note its URL for §06.
Tip Keep the library tidy — only template files belong here. The publicly displayed name of the library will be your organization's name.

04 Install the SharePoint Online Management Shell

Registration is done in PowerShell. Use a clean, current install.

  1. Uninstall any old version. If a previous SharePoint Online Management Shell is installed, remove it via Add or remove programs first.
  2. Download the latest. Get it from the official download link and install.

05 Connect to SharePoint

Authenticate as a SharePoint Administrator. Replace the URL with your own tenant admin endpoint.

# Connect to your SharePoint admin center Connect-SPOService -Url https://contoso-admin.sharepoint.com

06 Register the library as a template OAL

This is the moment the ordinary library from §02 becomes an organization assets library. Run Add-SPOOrgAssetsLibrary with the template type — it promotes the library, enables a delivery CDN, and surfaces your templates under the org tab in PowerPoint.

Before you run this — permission check The command will fail unless "Everyone except external users" already has Read on the Templates document library (§02 step C2). Confirm that first, or you'll get: "Everyone except external users" needs to be given read permission to the document library.

Syntax

Add-SPOOrgAssetsLibrary -LibraryUrl <URL> [-ThumbnailUrl <URL>] [-OrgAssetType OfficeTemplateLibrary] [-CdnType <Private or Public>]

Example — register a PowerPoint template library

Add-SPOOrgAssetsLibrary ` -LibraryUrl https://contoso.sharepoint.com/sites/branding/Templates ` -ThumbnailUrl https://contoso.sharepoint.com/sites/branding/Templates/logo.png ` -OrgAssetType OfficeTemplateLibrary
ParameterWhat it does
-LibraryUrlAbsolute URL of the document library holding your .potx files.
-ThumbnailUrlOptional. Image shown as the card background in the file picker. Must be on the same site. The card's display name is your org name.
-OrgAssetTypeOfficeTemplateLibrary for templates. (Omitting it defaults to an image library.)
-CdnTypeOptional — leave it off. Omitting it is fine for templates. (The manually configured private CDN is being deprecated, so don't set it unless you specifically need a public CDN.)
Note — CDN Adding an organization assets library will enable a content delivery network (CDN) for your organization to provide fast and reliable performance for shared assets. You'll be prompted to enable a CDN for each organization asset library you add. For more information, see Content Delivery Networks (CDNs).

07 How users find the templates

In PowerPoint, users open your organization's templates folder and pick a deck. The exact path differs by platform. Users need an Office 365 E3 or E5 license. Source: Use your organization's templates in PowerPoint.

PowerPoint for Windows

  • Select New on launch, or File → New.
  • Open the folder with your organization's name on it.
  • Right-click the template → Create.

PowerPoint for Mac

  • Select New on launch, or File → New from template.
  • Open the folder with your organization's name on it.
  • Double-click the template to create a presentation.

PowerPoint for the web E3 / E5

  • Go to Office.com → select the PowerPoint icon.
  • On the File tab: New → Browse templates.
  • On the left, select your company name, then pick the template.
Propagation Allow up to 24 hours for the library to appear to users in the desktop apps. Organization fonts are also supported in PowerPoint for the web.
Bonus Templates supported by Design Suggestions give users on-brand, coherent slide layouts automatically (Home → Design Suggestions).

08 Manage & update

Inspect, re-thumbnail, or remove libraries with the companion cmdlets.

# List all organization asset libraries on the site Get-SPOOrgAssetsLibrary # Update the thumbnail for a library Set-SPOOrgAssetsLibrary -LibraryUrl <URL> -ThumbnailUrl <URL> # Remove a library from the OAL designation Remove-SPOOrgAssetsLibrary -LibraryUrl <URL>

09 Gotchas & checklist

The short list of things that trip people up.

  • Wrong asset type. Templates need OfficeTemplateLibrary; the default is an image library.
  • Same-site rule. Every OAL (images and templates) must sit on one shared site — you can't scatter them.
  • 30-library cap. A single org supports up to 30 organization asset libraries total.
  • Must be .potx. PowerPoint templates must be .potx — a .pptx won't work. (Word uses .dotx, Excel .xltx.)
  • Web is license-gated. Without E3/E5, the org tab won't appear in PowerPoint for the web.
  • Patience. Up to 24 hours for desktop propagation — don't declare failure early.
  • Read access (the #1 failure). "Everyone except external users" must have Read on the document library before you register — granting it only on the site can fail the command. Users also need read on the org root site.
