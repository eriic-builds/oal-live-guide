Set up an Organization Asset Library for PowerPoint templates

Designate a SharePoint document library as an organization assets library (OAL) of type OfficeTemplateLibrary so your corporate .potx templates appear under your organization's tab when people create a new PowerPoint file.

Audience: SharePoint Administrator (or higher) Source: Microsoft Learn, Create an organization assets library

Built and grounded on official Microsoft documentation. Unofficial community resource, not affiliated with Microsoft.

What you get at the end

A user opens PowerPoint, clicks File → New, and your branded templates sit in a tab named after your organization. They pick one and start an on-brand deck. Same experience for every user, on desktop and web.

PowerPoint  →  File  →  New

The shortest path to that

  1. Build the site and library. One SharePoint site, one document library. (§02)
  2. Grant Read to "Everyone except external users". The step most people miss. (§02 C)
  3. Upload .potx templates. Not .pptx. (§03)
  4. Promote the library. One PowerShell command. (§06)
  5. Users open the gallery. File → New → Organization Templates. (§07)
What you will not find in the official docs The read grant goes on the document library, not the site. The path is Settings → Site permissions → Share site, set to Read. Templates must be .potx, not .pptx. Copilot searchable does not work on a template library.

Why this guide exists

You want every user in your organization to start new PowerPoint decks from approved, on-brand templates across the tenant. Users upload personal templates, but those templates stay private and never appear for other people. Some users see no organization templates option in PowerPoint. You need one supported way to push templates to everyone and show them the same way for each user.

Two features handle branding, and they work in parallel:

Organization Asset Library (OAL) this guide

Powers the native template gallery at File → New → Organization Templates in PowerPoint on desktop and web. Publishes org-level templates that every user sees, and adds the branded slide layouts they start from.

Brand Kit Templates

Set up in Copilot → Create → More → Brand kits. A brand manager publishes a kit to Official for org-wide access. A Brand Kit applies your brand theme, colors, fonts, and logos, to Copilot-generated and suggested designs. It does not surface .potx templates in the File → New gallery. That is the OAL's job.

Bottom line Brand Kits apply the theme. OAL adds the template layout. They run in parallel. For the File → New template gallery, you want the OAL. This guide sets up the OAL.

00 Overview

An organization assets library is a SharePoint document library you promote so its files surface across Microsoft 365. It holds different asset types. For branded decks you want the Office templates type. This guide covers the Office templates type.

Office templates this guide

When a user creates a new Office document, they see a tab for your organization with the available templates. Works in Word, Excel, and PowerPoint.

Images & logos

Photos and logos appear under "Your organization" in the SharePoint file picker and "Brand Images" in PowerPoint. This is a separate setup. This guide does not cover it.

Key parameter For templates, register the library with -OrgAssetType OfficeTemplateLibrary. If you omit the type, it defaults to an image library. An image library does not surface templates.

02 Build the SharePoint site from scratch

An organization assets library starts as an ordinary SharePoint document library. You promote it with PowerShell later. First you build a site and library. Then you turn it into an OAL. You do not need Brand Central. Source: Create an organization assets library.

A. Create a new site

  1. Open the create flow. Go to https://contoso.sharepoint.com and click + Create site on the SharePoint start page. Admins also use SharePoint admin center → Active sites → + Create.
  2. Choose a site type. Pick Communication site. Any modern site works, including communication, a Microsoft 365 Group team site, or a non-group team site. A communication site fits the "few publishers, everyone reads" pattern and does not create a Group you do not need.
  3. Pick a design (optional). The Topic or Blank template works. The site design has no effect on OAL behavior. It only styles the landing page.
  4. Name and address it. Use a site name like Brand Templates. The site address becomes the URL path, for example /sites/branding → https://contoso.sharepoint.com/sites/branding. Note this URL. You build on it.
  5. Set language and owner. Choose the default language. The default language is permanent. Set yourself as primary administrator, then Finish. The site provisions in a few seconds.

B. Create the document library

  1. Add a library. On the new site, click + New → Document library → Blank library.
  2. Name it. Use a name like Templates and an optional description. Click Create. Its URL becomes https://contoso.sharepoint.com/sites/branding/Templates. You register this library URL in §06.
  3. It must be a library, not a folder. The OAL designation accepts document libraries only. For sub-groupings, use folders inside this one library. Register each library separately if you create more than one.

C. Set permissions correctly

  1. Publishers = Members or Owners. ⚙ Settings → Site permissions. Add the people who upload and maintain templates as Members (edit) or Owners (full control).
  2. Readers = "Everyone except external users" (REQUIRED). This group must have Read on the library before you register. Missing this grant is the top reason Add-SPOOrgAssetsLibrary fails. See the error callout below. Exact path: on the Templates site, click ⚙ Settings → Site permissions → Share site, add "Everyone except external users", and set the permission level to Read, not Edit. Then save.
  3. Fine-tune if needed. You customize permissions on up to 100 files or folders in the library, for example to hold back a draft template, without breaking the OAL.
If you hit this error Add-SPOOrgAssetsLibrary : "Everyone except external users" needs to be given read permission to the document library. The read grant is missing. Fix: on the Templates site go to ⚙ Settings → Site permissions → Share site, add "Everyone except external users" set to Read, then re-run the §06 command. aka.ms/orgassets.
Site vs. OAL This site and library are not an organization assets library yet. Right now it is a normal SharePoint library. You apply the OAL designation later with Add-SPOOrgAssetsLibrary in §06. The designation surfaces the templates under your org tab in PowerPoint and enables a delivery CDN. §03 fills the library. §04 through §06 perform the promotion.

03 Build, save & upload your .potx templates

⚠ The file MUST be a .potx The Organization Asset Library supports PowerPoint templates in .potx format. A standard .pptx presentation is not a supported organizational template format and won't appear in the PowerPoint organizational templates experience. Always save and upload PowerPoint templates as .potx. Source: supported template formats.

A template is a deck whose slide master and layouts carry your branding. Save it in the .potx format, the only PowerPoint format the OAL accepts. Then upload it to your library. Source: Create and save a PowerPoint template.

Already have a template If you have a finished .potx, upload it in card C. No rebuild needed. If you have a polished .pptx, open it and run File → Save As → PowerPoint Template (*.potx) once in card B, then upload. Card A covers building a new template from scratch.

A. Build the template (new templates only)

  1. Start a blank deck. File → New → Blank Presentation.
  2. Edit the slide master. Go to View → Slide Master, then apply your brand with Themes, Background Styles, and Insert Placeholder on the master and its layouts. Editing the master and layouts makes it a reusable template.

B. Save as .potx

  1. Save As. File → Save As (or Save a Copy in Microsoft 365).
  2. Choose the template type. In the Save as type list, select PowerPoint Template. PowerPoint switches the location to your Custom Office Templates folder. Name it and Save.

C. Upload to the library

  1. Upload the .potx. Put the .potx files into the Templates library from §02. Check the extension reads .potx, not .pptx, before you register. You register this library next.
  2. (Optional) Add a thumbnail image. For a custom card background in the file picker, place an image, for example your logo, on the same site and note its URL for §06.
Tip Keep the library tidy. Only template files belong here. The public name of the library is your organization name.

06 Register the library as a template OAL

This step turns the library from §02 into an organization assets library. Run Add-SPOOrgAssetsLibrary with the template type. It promotes the library, enables a delivery CDN, and surfaces your templates under the org tab in PowerPoint.

Permission check before you run this The command fails unless "Everyone except external users" already has Read on the Templates document library from §02 step C2. Confirm this first, or you get: "Everyone except external users" needs to be given read permission to the document library.

Syntax:

Add-SPOOrgAssetsLibrary -LibraryUrl <URL> [-ThumbnailUrl <URL>] [-OrgAssetType OfficeTemplateLibrary] [-CdnType <Private or Public>]

Example, register a PowerPoint template library:

Add-SPOOrgAssetsLibrary ` -LibraryUrl https://contoso.sharepoint.com/sites/branding/Templates ` -ThumbnailUrl https://contoso.sharepoint.com/sites/branding/Templates/logo.png ` -OrgAssetType OfficeTemplateLibrary
ParameterWhat it does
-LibraryUrlAbsolute URL of the document library holding your .potx files.
-ThumbnailUrlOptional. Image shown as the card background in the file picker. Must be on the same site. The card's display name is your org name.
-OrgAssetTypeOfficeTemplateLibrary for templates. (Omitting it defaults to an image library.)
-CdnTypeOptional. This sets the CDN type, Public or Private, and only if you pass it. You do not need it for templates. The CDN itself is enabled at the confirm prompt, not here. See the CDN note below. The private type is being deprecated, so pass Public only if you need a public CDN.
Note - CDN Adding an organization assets library will enable a content delivery network (CDN) for your organization to provide fast and reliable performance for shared assets. You'll be prompted to enable a CDN for each organization asset library you add. For more information, see Content Delivery Networks (CDNs).

At the confirm prompt, type Y to enable it. Y is the default and the recommended choice. It caches your shared template assets closer to users for faster downloads. The Office 365 CDN is included in your SharePoint Online subscription.

07 How users find the templates

In PowerPoint, users open your organization's templates folder and pick a deck. The exact path differs by platform. Users need an Office 365 E3 or E5 license. Source: Use your organization's templates in PowerPoint.

PowerPoint for Windows

  • Select New on launch, or File → New.
  • Open the folder with your organization's name on it.
  • Right-click the template → Create.

PowerPoint for Mac

  • Select New on launch, or File → New from template.
  • Open the folder with your organization's name on it.
  • Double-click the template to create a presentation.

PowerPoint for the web E3 / E5

  • Go to Office.com → select the PowerPoint icon.
  • On the File tab: New → Browse templates.
  • On the left, select your company name, then pick the template.
Propagation Allow up to 24 hours for the library to appear to users in the desktop apps. Organization fonts are also supported in PowerPoint for the web.
Bonus Templates supported by Design Suggestions give users on-brand, coherent slide layouts automatically (Home → Design Suggestions).

08 Manage & update

Inspect, re-thumbnail, or remove libraries with the companion cmdlets. Run one at a time.

List all organization asset libraries on the site:

Get-SPOOrgAssetsLibrary

Update the thumbnail for a library:

Set-SPOOrgAssetsLibrary -LibraryUrl https://contoso.sharepoint.com/sites/branding/Templates -ThumbnailUrl https://contoso.sharepoint.com/sites/branding/Templates/logo.png

Remove a library from the OAL designation:

Remove-SPOOrgAssetsLibrary -LibraryUrl https://contoso.sharepoint.com/sites/branding/Templates

09 Gotchas & checklist

Review these before you finish.

  • Wrong asset type. Templates need OfficeTemplateLibrary. The default is an image library.
  • Same-site rule. Every OAL for images and templates sits on one shared site.
  • 30-library cap. A single org supports up to 30 organization asset libraries total.
  • Must be .potx. PowerPoint templates must be .potx. A .pptx does not work. Word uses .dotx, Excel uses .xltx.
  • Web is license-gated. Without E3 or E5, the org tab does not appear in PowerPoint for the web.
  • Propagation time. Desktop apps take up to 24 hours to show the library. Do not declare failure early.
  • Read access, the top failure. "Everyone except external users" must have Read on the document library before you register. Granting it on the site alone fails the command. Users also need read on the org root site.
